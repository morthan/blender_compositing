Edit Methods
^^^^^^^^^^^^

image:img/figure2_12_1.png[]

*Рисунок 2.12* 'Категория «Edit Methods»'

Группа *Material linked to* определяет, к чему привязывается материал при
создании нового объекта. Может быть выставлен один из двух режимов:

ObData::
    Материал привязывается к блоку `ObData` объекта.

Object::
    Материал привязывается к блоку объекта.

Группа *Add new objects* определяет действия, которые выполняются при добавлении
нового объекта.

Если включена кнопка *Switch to Edit Mode*, то после добавления нового объекта
Blender автоматически переключится в режим редактирования (изменения формы этого
объекта).

Если включена кнопка *Aligned to View*, то новый объект при добавлении будет
автоматически развёрнут «лицом» к наблюдателю. По умолчанию, когда эта кнопка
отключена, при добавлении объекта *Plane* (плоскость), он будет располагаться в
плоскости `X-Y` независимо от того, с какой стороны вы на неё смотрите. Если же
кнопка включена и вы переключитесь, к примеру, на вид из камеры, то плоскость
при добавлении будет развёрнута к вам «лицом». По идее, эта опция может
пригодиться при работе с проекционными плоскостями.

В группе *Transform* находится всего одна кнопка -- *Drag Immediately*. Она
заведует перемещением объектов с помощью вышеупомянутого виджета в виде трёх
стрелочек и по умолчанию выключена. Таким образом, когда вы перемещаете объект с
помощью мыши, вам требуется в итоге подтвердить операцию щелчком по левой кнопке
мыши -- тогда объект останется на том месте, куда вы его перенесли. Если вместо
этого нажать правую кнопку мыши, то перемещение будет отменено и объект вернётся
туда, откуда вы его взяли.

Если же будет нажата кнопка *Drag Immediately*, то подтверждать перенос не надо:
просто перетаскиваете объект и отпускаете там, где нужно. Плюс -- меньше времени
уходит, да и на тачскрине работать удобнее. Минус -- отменить перемещение можно
только с помощью *Ctrl+Z*.

Кстати, о *Ctrl-Z*. Группа *Undo* заведует именно отменой неправильных действий.
В неё входят следующие контролы:

Steps::
    Здесь указывается размер истории отмен, то есть количество операций, которые
    можно отменить. Чем больше размер истории, тем больше памяти на это нужно.

Memory limit::
    Максимальный объём памяти (в мегабайтах), выделяемой под историю отмен. По
    умолчанию стоит `0`, означающий, что контроль объёма памяти вообще
    отсутствует.

Global Undo::
    Эта кнопка включена по умолчанию и означает, что при каждом действии в
    истории отмен запоминается 'полная копия' текущего состояния файла. Если её
    выключить, то будут запоминаться только 'различия' между версиями. Как по
    мне, первый вариант надёжнее, а то мало ли...

Следующая группа контролов, *Auto Keyframe*, отвечает за автоматическую
расстановку ключевых кадров при анимацииfootnote:[Я не пользуюсь автоматической
расстановкой ключей и, соответственно, не могу ничего о ней рассказать,
извините.] и содержит следующие кнопки:

Auto-Keying Enabled::
    Включает режим автоматической расстановки ключевых кадров. По умолчанию
    выключена, но если вы гуру, принципиально не делающий ошибок, то можете
    включить.

Available::
    Автоматически вставляет ключевые кадры в доступные кривые.

Needed::
    Вставляет ключевые кадры только в том случае, если они нужны.

Use Visual Keying::
    Для объектов с ограничителями ('constrained objects') использовать
    визуальную расстановку ключевых кадров.

Группа контролов *Grease Pencil* управляет, как несложно догадаться, одноимённым
инструментом -- чем-то вроде маркера, позволяющего рисовать прямо по 3D-объектам
или отрендеренным кадрам видео. Польза от него бывает только в том случае, когда
нужно нарисовать корявую стрелочку, показывающую на криво отрендеренную часть
картинки. Поскольку я с ним не работал никогда, то о сути настроек не могу даже
догадываться.

Группа *Duplicate with object* определяет, что дублируется при дублировании
объекта:

Mesh::
    Форма объекта -- набор точек, соединённых рёбрами.

Surface::
    Поверхность, сформированная из кривых NURBS.

Curve::
    Кривые.

Text::
    Текстовые объекты.

Metaball::
    Мета-объекты (сгустки).

Armature::
    Арматура (скелет, опорно-двигательная система).

Lamp::
    Источник освещения.

Material::
    Материал.

Texture::
    Текстура.

Ipo::
    Кривые анимации.

По умолчанию включены все пункты, кроме материала, текстуры и кривых анимации.
То есть при дублировании любого объекта эти три параметра на новый экземпляр не
переносятся и должны назначаться заново.
