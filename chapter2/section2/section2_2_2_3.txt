Фоновая картинка
^^^^^^^^^^^^^^^^

При некоторых операциях очень полезно иметь под рукой изображение, с которым
можно сверяться. В случае моделирования это могут быть чертежи, позволяющие
наложить смоделированную деталь на одну из проекций в чертеже и проверить
правильность моделирования. Если же мы совмещаем несколько изображений (в чём и
состоит задача композитинга), то очень удобно иметь под рукой фоновую картинку,
чтобы контролировать правильность совмещения.

В Blender есть возможность отобразить в окне 3D-вида фоновую картинку. Для этого
нужно выбрать пункт меню *View* -> *Background Image*. После этого на экране
появится окно свойств фонового изображения с одной-единственной кнопкой *Use
Background Image*. Если эту кнопку нажать, в окне появятся дополнительные
контролы, из которых нас пока интересует только кнопка *Load*. Нажатие на неё
позволяет загрузить фоновую картинку, найдя на диске её файл.

После того, как фоновая картинка будет загружена, окно приобретёт вид,
показанный на рисунке 2.17.

image:img/figure2_17.png[]

*Рисунок 2.17* 'Окно свойств фоновой картинки'

Ползунок *Blend* регулирует прозрачность фоновой картинки в процентах. При *0*
картинка непрозрачна, при *50* -- прозрачна наполовину, при *100* -- прозрачна
полностью (то есть не видна совершенно).

Поле *Size* отвечает за размер картинки, то есть во сколько раз её увеличивать.

Поля *X Offset* и *Y Offset* задают смещение картинки по горизонтали и вертикали
соответственно.

Ниже находится ряд кнопок, задающих тип фонового изображения:

Still::
    Отдельная статическая картинка.

Movie::
    Видеофайл. При прокрутке анимации в Blender, фоновая картинка будет
    изменяться, отображая всякий раз новый кадр видео.

Sequence::
    Последовательность изображений. Полностью аналогично предыдущему пункту, но
    видео содержится не в одном видеофайле, а в наборе пронумерованных
    файлов-картинок.

Generated::
    Помощь утверждает, что это «сгенерированное изображение». Есть вероятность,
    что оно берётся из каких-то внутренних каналов Blender, допустим, из
    рендеринга другой сцены. На практике я не встречался с использованием этой
    опции.

Ниже располагается группа из четырёх кнопок, задающих опции отрисовки некоторых
особенностей картинки:

Fields::
    Имеет смысл для чересстрочного видео. Включение этой опции выключает
    «гребёнку» движущихся кадровfootnote:[Подробности будут объясняться потом, в
    одной из следующих глав.].

Odd::
    Имеет смысл для чересстрочного видео. Изменяет порядок вывода полукадров
    (они могут начинаться либо с чётного числа, либо с нечётного).

Anti::
    Включает режим сглаживания краёв (по-английски -- 'anti-aliasing'). Работает
    только для однотонных изображений.

Premul::
    Переключает отображение прозрачности в предумноженный режимfootnote:[О том,
    какие бывают режимы прозрачности, см. главу про работу с изображениями.].

Для статической картинки больше опций нет. Но, если мы выберем тип *Movie* или
*Sequence*, то появятся дополнительные опции, характерные для видео:

image:img/figure2_18.png[]

*Рисунок 2.18* 'Настройки фонового видео'

Кнопка *Auto Refresh* включает синхронизацию анимации Blender и фонового видео.
Обязательно нажмите её, если используете фоновое видео!

В поле *Frames* указывается количество кадров, которые будут использованы в
фоновом видео. То есть, это длина отрезка видео, только заданная не в секундах,
а в кадрах.

Поле *Offs* указывает, начиная с какого кадра брать видео для фоновой картинки.
Его задание позволяет брать отрезок для фонового видео не с начала а, допустим,
с середины ролика.

Поле *File/Img* имеет смысл при работе с чересстрочным видео (при включенной
опции *Fields*). Здесь указывается, сколько физических кадров одного видео или
изображения должны быть чересстрочными для получения цельного изображения.

Поле *StartFr* задаёт номер кадра Blender, начиная с которого будет
воспроизводиться фоновое видео.

Кнопка *Cyclic* зацикливает фоновое видео, заставляя его воспроизводиться снова
и снова.

[WARNING]
Обратите внимание, что фоновое изображение или видео показывается 'только' в
окне 3D-вида Blender! В готовой отрендеренной картинке/анимации его не будет.
Добавление фоновых изображений в результат рендеринга делается в другом месте и
объясняется позже.
